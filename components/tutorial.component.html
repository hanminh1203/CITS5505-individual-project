HTML
1. What is HTML?

HTML (Hyper Text Markup Language) was originally defined as a type of SGML in 1990, by Tim Berners-Lee.
HTML5 was a significant revamp of the language released in 2008.

The HTML5 Philosophy
Interoperability
Pages should be renderable on a wide variety of browsers.
Graceful Error Recovery
Small errors should not stop the page from rendering.
Backwards Compatibility
New features should not break the web.
Prioritize Users
The needs of the user come before designers, browsers, or theorists.
Separation of Concerns
HTML describe the type of information, not how it displays.
2. Document Structure

An HTML document is a tree data structure made of elements. Most elements consist of an opening tag, content, and a closing tag.

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>Welcome to Agile Web!</h1>
    <p>Ready to build a website?</p>
  </body>
</html>

Visible elements, like headings and paragraphs, are placed within the <body> branch, while metadata and the page title reside in the <head>.

3. Text and Semantic Content

HTML provides various tags to define the meaning of text:

  • <h1> to <h4>: Headings of varying importance.
  • <p>: Paragraphs.
  • <code>: Computer code snippets.
  • <em>: Emphasized (italic) text.

4. Images and Hyperlinks

All HTML tags can have a list of attributes which appear between its name and the right bracket of the opening tag.

Images

Images are inserted via the <img/> tag using the src attribute. The alt attribute provides text for if the image can't be displayed or for screen readers. 

<img src="https://imgs.xkcd.com/comics/university_website.png" alt="Pointed advice on university website design" style="width:500px">
Hyperlinks

Links are created using the anchor tag <a> with an href (hypertext reference) attribute.

5. Organizing Data: Lists and Tables
Lists

HTML supports Ordered Lists (<ol>) for numbered items and Unordered Lists (<ul>) for bullet points. Items are defined using <li>.

Tables

Tables are matrices of cells consisting of rows (<tr>), headers (<th>), and data cells (<td>).

6. Layout and Grouping

To group elements for formatting or layout, HTML uses Block and Inline containers:

  • <div>: A multi-line block used for larger sections.
  • <span>: Groups content on a single line, like words in a sentence.

HTML5 also introduced Advanced Layout Elements like <header>, <nav>, <section>, <article>, <aside>, and <footer> for better document structure.

7. Forms and Interaction

The <form> element is the standard way to send data from the browser to a server .

<form action="/submit-data" method="get">
  <label for="user-email">Email:</label>
  <input type="email" id="user-email" name="email" required>
  <input type="submit" value="Send">
</form>

Using correct type attributes (e.g., email, tel, date) allows browsers to provide specialized keyboards and native validation.

CSS (Cascading Style Sheets)

1. What is CSS?

CSS stands for Cascading Style Sheets
  • a stylesheet language for the web
  • used to specify the presentation (layout and style) of markup languages
  • can be applied to any XML document as well as HTML
  • superseded many HTML attributes that mixed presentation with content
Advantages for the web:
Speed
Stylesheet(s) downloaded once, rather than with each page (if content and style information is intermingled)
Maintainability
Can be "centrally" maintained, easier to update
Accessibility
Keeping style in CSS and content in HTML maintains semantic markup and improves accessibility for assistive technologies
Portability
Consistent styling across all devices supporting browsers.
Reduced work
e.g., don't have to specify alignment for every element.
Consistency
make an organisation's web pages have a consistent "look and feel" that matches the corporate ID, brand.

2. Applying CSS to Your Documents

There are three primary ways to include CSS in your project:

A. Inline Styles

Applied directly to a single tag using the style attribute. While useful for debugging, they generally defeat the purpose of uniform styling.

<p style="color: blue; font-size: 16px;">This is an inline style.</p>

B. Document (Internal) Style Sheets

Contained within a <style> tag inside the HTML <head>. These apply rules to the entire document.

<style>
  p { color: red; }
</style>

C. External Style Sheets

Separate .css files linked in the <head>. This is the most efficient way to apply styles to many pages at once.

<link rel="stylesheet" type="text/css" href="styles.css">

3. Selectors: Targeting Elements

A selector determines which HTML elements a style rule applies to.

  • Universal Selector (*): Matches all elements on the page.
  • Element Selector: Matches a specific tag type (e.g., p, h1).
  • Class Selector (.class): Groups arbitrary sets of elements using the class attribute.
  • ID Selector (#id): Targets a unique element with a specific id attribute.
  • Pseudo-class (:state): Selects elements based on state, such as :hover or :visited.

4. The Box Model

Every element is laid out as four boxes that control spacing and borders.

  1. Content: Where the actual element (text/image) is rendered.
  2. Padding: The space between the border and the content.
  3. Border: The line surrounding the padding and content.
  4. Margin: The transparent space between the border and neighboring elements.

5. Common Properties

Fonts and Text

  • font-size: Sets the size using numbers or names (e.g., xx-large) .
  • text-align: Controls alignment (left, center, right, or justify).
  • text-decoration: Adds effects like underline or line-through.

Colors

Colors can be specified by 140 supported names, a set of 16 standard colors, or 16 million different hexadecimal codes (e.g., #FFFFFF for white).

6. Layout and Conflict Resolution

By default, elements follow normal flow, appearing sequentially above each other . This can be overridden using:

  • Float: Shifts elements to the left or right, allowing text to flow around them .
  • Position: Offers static, relative, absolute, fixed, and sticky modes using offsets like top and left.

Dealing with Conflicts (The Cascade)

When multiple rules apply to one element, browser precedence rules take over. Generally, priority follows:

Inline Styles > ID Selectors > Class/Attribute Selectors > Element Selectors

The !important modifier can be used to override these standard priorities.

7. Modern Styling with Frameworks

Manually laying out pages with raw CSS is difficult and must account for many device sizes. CSS Frameworks like Bootstrap are open-source libraries that provide pre-built, responsive components. Bootstrap uses a 12-column grid system to align content easily across mobile, tablets, and PCs.
